A) HTML Structure - top

This theme features a responsive layout designed within a container element. Inside the container, there's a row element containing a column set to occupy the full width (col-md-12) with centered text. Within this column, there's a heading (h2) with a specific class "section-heading mergecolor" and a paragraph with class "section-subheading mergecolor". These elements provide information about choosing an available RDP plan and selecting a data center location.

Following this section, there's a set of tabs for different plan options. Each tab has a header group with list items, and below that, there's another set of tabs for selecting billing periods. Both sets of tabs are contained within a row element.

The content of the tabs is organized within a div with class "tabs-content". Each tab item has a class "tabs-item active show". Inside each tab, there's a slick slider container with plan information. Each plan is presented within a "plan-container" div with an id indicating the header (e.g., header1plan1). Inside these containers, there's detailed information about the RDP plans, including location, CPU, disk, data, and RAM specifications, along with pricing and an order button.

The structure repeats for a total of ten tabs, each representing different plan options.

This theme is structured to provide a user-friendly interface for selecting among various RDP plans and billing periods.

This theme follows a fixed layout structure with two columns. All information within the main content area is enclosed within a div identified by the id "primaryContent". The sidebar's content, corresponding to column #2, is nested within a div marked with the id "secondaryContent". The "configsection.json" file contains specifications for period headers and headers. It allows for easy customization of values, visibility, pricing, links, and stockout status:

Period Headers:

Each period header is defined with attributes such as "name", "value", and "visible". Users can modify the "value" field to change the text displayed for each period header. The "visible" attribute can be toggled to control the visibility of each period header.

Headers:

Headers are categorized with a unique identifier under the "name" attribute. Each header includes details about multiple plans within an array. For each plan, users can adjust various parameters such as name, stockout status, pricing for different billing periods (monthly, quarterly, half-yearly, yearly), and associated links for ordering. Prices for different billing periods (monthly, quarterly, half-yearly, yearly) can be updated as required. Links for ordering can be modified to direct users to appropriate destinations. The "stockout" attribute can be modified to reflect the availability status of each plan, allowing users to mark plans as out of stock if necessary.

This configuration file offers a user-friendly approach for customizing headers, period headers, pricing, links, and stockout status, ensuring flexibility and ease of management within the theme.

B) CSS Files and Structure - top

In this theme, two CSS files are utilized for styling:

The first CSS file, "bootstrap.min.css", serves as a generic reset file. It assists in ensuring a consistent interpretation of HTML elements across various browsers by overriding default behaviors. Additionally, this file includes some general styling rules such as anchor tag colors, font sizes, and other basic styling attributes. It's important to note that these values may be overridden elsewhere in the file or by subsequent CSS files.

The second CSS file, "main-style.css", contains all specific stylings tailored for the page. This file is organized into sections, allowing for better organization and maintenance of the CSS code.

Additionally, the Font Awesome library is included via a CDN link to provide access to a wide range of iconography for use within the theme.

If you would like to edit a specific section of the site, simply find the appropriate label in the CSS file, and then scroll down until you find the appropriate style that needs to be edited.

    Function to update the DOM dynamically:
    
	$.ajax({
	url: "js1/configsection.json",
	dataType: "json",
	success: function (jsonData) {
	  data = jsonData;
	  var headerMatched = false;

	  if (data && data.headers) {
		data.headers.forEach(function (section) {
		  if (section.name === selectedheader) {
			for (var i = 0; i < 5; i++) {
			  $("#" + selectedheader + "-price" + (i + 1)).html(
				"$" +
				  section.plans[i][price] +
				  '/mo'
			  );
			  $("#" + selectedheader + "-link" + (i + 1)).attr(
				"href",
				section.plans[i][link]
			  );
			}
			headerMatched = true;
		  }
		});
	  }

	  if (headerMatched) {
		$("#" + period)

	},
